## Только один package на каталог
В исходных файлах Go может быть только один package на каталог. Объяснение на эту тему: https://dave.cheney.net/2014/12/01/five-suggestions-for-setting-up-a-go-project.

## Доступность на основе регистра (public/private)
В Go общедоступные функции и структуры начинаются с заглавной буквы, а приватные — со строчной. Если не хотим, чтобы внутренняя работа нашего алгоритма была доступна всем: пишем название со строчной буквы.

## Срезы (slices) и массивы (massive)
Срезы - это постоянно расширяющиеся массивы. В спецификации всё выглдядит так:
type slice struct {
    ptr *T   // указатель на массив
    len int  // длина
    cap int  // ёмкость
}

Из этого же следует, что если попытаться присвоить значения массива в срез, то при изменении среза, исходный массив также будет изменён например:

x := [3]string{"Лайка", "Белка", "Стрелка"}
y := x[:] // slice "y" ссылается на исходный массив "x"

y[1] = "Belka" // значение под 1-мы индексом сейчас "Belka", это значение именилось и в "y", и в "x"

fmt.Printf("%T %v\n", x, x)
fmt.Printf("%T %v\n", y, y)

Из-за этого лучше создавать не ссылку на исходный массив, а делать его копию:

x := [3]string{"Лайка", "Белка", "Стрелка"}
z := make([]string, len(x))
copy(z, x[:])

Тогда мы получим несколько коллекций, которые не зависят друг от друга и их можно именять

# Нейминг:
## Для тестов
(https://quii.gitbook.io/learn-go-with-tests/go-fundamentals/hello-world#writing-tests)

Написание теста похоже на написание функции, но с некоторыми правилами

-  Он должен находиться в файле с таким названием, как xxx_test.go

- Тестовая функция должна начинаться со слова Test

- Функция тестирования принимает только один аргумент t *testing.T

- Чтобы использовать тип *testing.T нужно выполнить в начале файла с тестами import "testing"

## Для примеров функций
Примеры функций начинаются с Example (так же, как тестовые функции начинаются с Test), и они находятся в _test.go-файлах пакета.

Примеры нужны не только для чтения документации, но и для провреки документации на актуальность: каждый раз когда запускаются тесты, идёт компиляция кода примера и если эта компиляция не проходит - значит были какие то изменения в вызываемой в примере фунукции. Значит, пример неактуален
